# UI Specification

### 전체 레이아웃
```
+----------------------------------------------------------+
|  Header (로고, 알림, 프로필)                               |
+------------+---------------------------------------------+
|            |                                             |
|  Sidebar   |              Main Content                   |
|            |                                             |
|  - 대시보드  |                                             |
```

## 1. 대시보드 (Dashboard)

### URL
`/` 또는 `/dashboard`

### 구성 요소

#### 1.1 오늘 요약 카드
```
+----------------------------------+
|  📅 오늘 - 2024년 1월 1일 (월)     |
|                                  |
|  할 일 5개 중 2개 완료             |
|  [=====>-----] 40%               |
|                                  |
|  [오늘 보기 →]                    |
+----------------------------------+
```

#### 1.2 이번 주 진행률
```
+----------------------------------+
|  📊 이번 주 진행률                 |
|                                  |
|  월 [====] 80%                   |
|  화 [==  ] 40%                   |
|  수 [    ] 0%    ← 오늘          |
|  목 [-   ] 예정                   |
|  금 [-   ] 예정                   |
|                                  |
+----------------------------------+
```

#### 1.3 다가오는 일정
```
+----------------------------------+
|  ⏰ 다가오는 일정                  |
|                                  |
|  14:00 프로젝트 미팅 (30분 후)     |
|  16:00 코드 리뷰                  |
|                                  |
+----------------------------------+
```

#### 1.4 빠른 액션
```
+----------------------------------+
|  [+ 할 일 추가]  [📋 계획 수립]    |
|  [📊 회고 보기]                   |
+----------------------------------+
```

## 3. 주간 계획 (Weekly Planning)

### URL
`/planning` 또는 `/week`

### 구성 요소

#### 3.1 주간 헤더
```
+----------------------------------------------------------+
|  [< 이전 주]  2024년 1월 1일 ~ 1월 7일  [다음 주 >]        |
|                                                          |
|  상태: [DRAFT]                              [✓ 확정하기]  |
+----------------------------------------------------------+
```

#### 3.2 주간 캘린더 뷰
```
+--------+--------+--------+--------+--------+--------+--------+
|   월   |   화   |   수   |   목   |   금   |   토   |   일   |
|  1/1   |  1/2   |  1/3   |  1/4   |  1/5   |  1/6   |  1/7   |
+--------+--------+--------+--------+--------+--------+--------+
|        |        |        |        |        |        |        |
| ☐ 운동 | ☐ 운동 | ☐ 운동 |        | ☐ 운동 |        |        |
|        |        |        |        |        |        |        |
| ☐ 미팅 | ☐ 리뷰 |        | ☐ 발표 |        |        |        |
|        |        |        |        |        |        |        |
| ☐ 정리 |        | ☐ 문서 |        | ☐ 회고 |        |        |
|        |        |        |        |        |        |        |
| [+ 추가]| [+ 추가]| [+ 추가]| [+ 추가]| [+ 추가]| [+ 추가]| [+ 추가]|
+--------+--------+--------+--------+--------+--------+--------+
```

#### 3.3 드래그 앤 드롭
- Task를 다른 날짜로 드래그하여 이동
- CONFIRMED 상태에서 이동 시 자동으로 ChangeLog 기록

#### 3.4 확정 확인 모달
```
+------------------------------------------+
|  계획을 확정하시겠습니까?             [X]  |
+------------------------------------------+
|                                          |
|  확정 후에는 모든 변경사항이 기록됩니다.   |
|  주말에 어떻게 변경되었는지 확인할 수      |
|  있습니다.                               |
|                                          |
|  총 15개의 할 일이 계획되었습니다.        |
|                                          |
|  [취소]                        [확정]    |
+------------------------------------------+
```

## 4. 주간 회고 (Weekly Review)

### URL
`/review` 또는 `/review/{weekStartDate}`

### 구성 요소

#### 4.1 회고 헤더
```
+----------------------------------------------------------+
|  📊 주간 회고: 2024년 1월 1일 ~ 1월 7일                    |
+----------------------------------------------------------+
```

#### 4.2 통계 요약
```
+----------------------------------------------------------+
|  +------------+  +------------+  +------------+           |
|  |    20     |  |    15     |  |   75%     |           |
|  |  총 계획   |  |   완료    |  |  완료율   |           |
|  +------------+  +------------+  +------------+           |
|                                                          |
|  +------------+  +------------+  +------------+           |
|  |     2     |  |     3     |  |    12     |           |
|  |   취소    |  |   연기    |  |  총 변경   |           |
|  +------------+  +------------+  +------------+           |
+----------------------------------------------------------+
```

#### 4.3 일별 완료율 차트
```
+----------------------------------------------------------+
|  일별 완료율                                              |
|                                                          |
|  100% |         ■                                        |
|   80% |    ■              ■                              |
|   60% |                        ■                         |
|   40% |         ■                   ■                    |
|   20% |                                                  |
|    0% +----+----+----+----+----+----+----                |
|        월   화   수   목   금   토   일                   |
+----------------------------------------------------------+
```

#### 4.4 변경 유형별 통계
```
+----------------------------------------------------------+
|  변경 유형 분석                                           |
|                                                          |
|  상태 변경      ████████████████  15                     |
|  시간 변경      ████  3                                  |
|  날짜 이동      ████  3                                  |
|  추가          ██████████  5                             |
|  삭제          ██  1                                     |
+----------------------------------------------------------+
```

#### 4.5 변경 이력 타임라인
```
+----------------------------------------------------------+
|  변경 이력                           [필터: 전체 ▼]       |
+----------------------------------------------------------+
|                                                          |
|  1월 5일 (금)                                            |
|  ├─ 16:30  "코드 리뷰" 완료 처리                         |
|  └─ 14:00  "프로젝트 미팅" 시간 변경 (14:00 → 15:00)     |
|            사유: 회의실 변경                              |
|                                                          |
|  1월 4일 (목)                                            |
|  ├─ 18:00  "보고서 작성" 완료 처리                       |
|  └─ 09:00  "운동" 금요일로 이동                          |
|            사유: 아침에 급한 일정                         |
|                                                          |
|  1월 3일 (수)                                            |
|  └─ 10:00  "긴급 버그 수정" 추가됨                       |
|                                                          |
|  [더 보기]                                               |
+----------------------------------------------------------+
```

#### 4.6 가장 많이 변경된 Task
```
+----------------------------------------------------------+
|  🔄 가장 많이 변경된 할 일                                 |
|                                                          |
|  1. "프로젝트 미팅" - 4회 변경                            |
|  2. "운동" - 3회 변경                                    |
|  3. "코드 리뷰" - 2회 변경                               |
+----------------------------------------------------------+
```

## 10. 상호작용 패턴

### 10.1 Task 체크박스
- 클릭 즉시 상태 변경 (낙관적 업데이트)
- 완료 시 strikethrough + 색상 변경
- 실패 시 롤백 + 에러 토스트

### 10.2 인라인 편집
- 제목 더블클릭 → 인라인 input으로 변경
- Enter로 저장, Esc로 취소
- blur 시 자동 저장

### 10.3 드래그 앤 드롭
- Task 드래그 시 반투명 + 그림자
- 유효한 드롭 영역 하이라이트
- 드롭 후 즉시 반영 + API 호출

### 10.4 토스트 알림
- 성공: 우하단, 초록색, 3초 후 자동 닫힘
- 에러: 우하단, 빨간색, 수동 닫기
- 정보: 우하단, 파란색, 5초 후 자동 닫힘
